## Critical Issues

### 1. Android Signing Key Committed to Git

- **File:** `pillage-first.jks` (root)
- **Risk:** Anyone with repo access can sign APKs as the app. The Android `.gitignore` has `*.jks` rules **commented out** (`apps/web/android/.gitignore:56-58`).
- **Fix:** Remove from git history, uncomment the gitignore rule, use CI secrets for signing.

### 2. `scripts/` Directory Gitignored but Referenced by `package.json`

- **File:** `package.json:18-22` references `scripts/inject-graphics.ts`, `scripts/remove-deprecated-directories.ts`, etc.
- **Problem:** `.gitignore:34` excludes `scripts/`. **The repo is broken for new contributors** — these scripts will fail on clone.
- **Fix:** Either commit the scripts or remove the npm script references.
